## 2. 코드에서 읽히는 설계 의도

| 설계 | 근거 |
|---|---|
| **32미션 전부 순차 진행** | `MISSIONS[S.mi]` — 한 번에 하나. 자유 탐색 없음. 방탈출이라기보다 **시련의 행렬**이다. |
| **`game` 29 : `quiz` 3** | 압도적으로 **장치 조작** 중심. 텍스트 퀴즈는 각인을 주는 결정적 순간(3회)에만. |
| **실패 = 리셋** | 대부분의 미션이 한 번 틀리면 처음부터. 난이도가 아니라 **긴장을 유지하기 위한 장치**. 실패 메시지가 전부 서사로 쓰여 있는 게 그 증거. |
| **힌트를 자원화** | "불씨" — 쓸수록 줄고 엔딩에 기록된다. 힌트 사용 자체를 플레이어의 **선택**으로 만들었다. |
| **각인 4개 = 최종 코드** | [9]→[14]→[22]→[24] 에서 각각 하나씩. **여정 전체를 기억해야 문이 열린다.** 4자리 코드 `4729`. |
| **엔딩 분기가 마지막 "미션"** | [32]는 자물쇠가 아니라 선택이다. *"이것은 당신의 이야기다."* — 힌트도 없다. |
| **진 엔딩을 문답으로 미리 흘린다** | [27] 이그나로크의 마지막 말: *"가라. 단, 문 앞에서 다시 생각하게 될 것이다."* → [32]의 선택 예고. |

## 3. 서사와 미션의 결합

이 게임의 강점은 **미션 메커니즘이 곧 장면**이라는 점이다. 수정 시 이 결합을 깨지 말 것.

| 미션 | 메커니즘 | 그게 곧 장면인 이유 |
|---|---|---|
| [13] 용의 눈을 피해 | 눈이 감겼을 때만 게이지 채우기 | 잠든 용 앞을 기어가는 **그 행위 자체** |
| [17] 숨의 박자 | 2.2초 주기 정점에 맞춰 8회 | 독안개 속에서 **숨을 고르는 동작** |
| [23] 심장의 박동 | 600ms 리듬 16회 | 훔친 심장과 **박동을 맞추는 것** |
| [27] 용과의 문답 | 지금까지 본 것들이 답 | 게임 전체가 **이 문답의 예습**이었다 |
| [32] 새벽의 선택 | 버튼 두 개 | 메커니즘이 없는 게 곧 메시지 |

## 4. ⚠️ 발견된 문제 / 개선 후보

### (가) BGM이 전혀 없다
[23] 심장의 박동은 **소리 없는 리듬 게임**이다. 화면만 보고 600ms 주기를 맞춰야 한다.
→ `03_image_bgm_guide.md` §6. **청각화 최우선.**

### (나) [30] "네 개의 각인" 의 안내가 불친절하다
각인의 수(4·7·2·9)는 **획득 순간 토스트로만** 알려준다. 적어두지 않은 플레이어는 되돌아갈 수 없다.
코드에도 이를 의식한 흔적이 있다 — 조각이 모자라도 *"모든 각인은 여정 중에 이미 얻었다"* 라는 안내가 뜬다.
→ **수첩/기록 UI를 하나 두는 것을 권장.** (다른 게임의 "보물 수첩" / `notesLabel` 패턴 참고)

### (다) registry.json 은 정확하다
`puzzles: 32` — 실측과 일치. (`magic-school`·`your-letter` 는 0으로 잘못 등재돼 있다.)
