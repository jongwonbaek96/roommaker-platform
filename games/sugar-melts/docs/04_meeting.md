## 안건 판정

- **안건① [조작 방식: 드래그 vs 탭]** → **채택**. 퀴즈 설계의 전 슬롯 탭/길게 누르기/토글 재설계를 정본으로 승인한다. 모바일 대응·단일 HTML+JS 구현이라는 플랫폼 제약(룸메이커 배포 환경)이 스토리의 "드래그 기반" 상정보다 우선한다. 스토리 §8-4 문구는 "탭 기반 기본, 드래그는 선택적 보조 입력(구현 시 지원 가능하면 병행 허용)"으로 수정할 것.
- **안건② [S14 챕터 표기 vs 완결 시점 데드락]** → **채택**. S14 2단계 분리(Ch4 관찰 기록 / T5 후 정렬 완결)는 자유 이동형 구조에서 유일하게 안전한 해법이다. 챕터 개방 조건은 "챕터 내 전 슬롯 클리어"가 아니라 **"핵심 슬롯 클리어"** 로 정의한다 — 구체적으로: Ch5 개방 조건에서 S14 완결 제외(S11·S12·S13 클리어로 개방), Ch6 개방은 S13의 사다리 열쇠 획득으로 충분. 스토리 슬롯표 S14 비고란에 2단계 구조를 명기해 두 문서를 동기화할 것.
- **안건③ [전시물 12점 목록 확정]** → **조건부 채택**. 목록 확정 자체는 승인하며 Ch1 체크리스트에 동일 목록 사용을 지시한다. 단, 부록 처리의 **근거 산술이 틀렸다**: 조약돌 주머니를 12점에 포함해도 위조는 7개가 아니라 6개(오븐·장부·문틀눈금·우리·뼈피리·주머니)로, 스토리 본문 "12개 중 6개 교체"와 정확히 일치한다. 따라서 **권장안: 조약돌 주머니를 12점 정식 목록에 포함하고, 비위조 항목 1개(레시피 카드 권장 — S28 판정에서 아무 역할 없음)를 목록에서 제외**하면 스토리 본문 수정 없이 정합된다. 부록 별항 구조를 유지하고 싶다면 스토리 본문을 "전시물 12점과 부록 1점, 그중 6개 교체"로 고쳐야 하며 이 경우 스토리 수정 1건이 추가된다. 구현 에이전트는 권장안을 기본값으로 적용할 것.

## 수정 지시

1. **[대상: 스토리+퀴즈] S30/S32/S33 — 장부 아이템 충돌 (치명)** — S30에서 "타다 만 장부 반쪽"을 가마 앞에 소모(안치)한 뒤 S33 소지품에 같은 물건이 재등장 → E1 진행 불가. **수정 방향**: S32 완료 시 정원사(헨젤) 강림 이벤트에서, 헨젤이 안치된 반쪽을 집어 플레이어가 보유한 출납 장부 반쪽(S08)과 **합쳐 "온전한 장부"로 건네는 연출을 추가**한다(대사 불요, 무언 전달 — 헨젤의 침묵 설정 유지). S33 소지품 표기를 "합쳐진 장부"로, 부속표 1에 "합쳐진 장부 | S32(수취) | S33·E1 | 선택 소모" 행을 추가. 스토리 §7 E1의 "장부와 발굴 기록 제출"은 그대로 유효해짐. 서사적으로도 "두 반쪽이 하나가 되어야 진실이 성립"하는 강화 효과.
2. **[대상: 퀴즈] S09 — 막대(―)=5 환산 단서 부재** — 소금 행 낙서는 ⦁=1과 화살표 방향만 확립하고, 정답(3배) 도출에 필수인 "―=5"는 힌트2에만 존재. **수정 방향**: 장부에 검산 가능한 행 1개 추가 — 예: "보리 ▸―⦁ 옆 낙서 '여섯 자루 들임'" (5+1=6으로 ―=5가 유일해). 힌트2는 유지.
3. **[대상: 퀴즈] S29 — "마지막 세 수" 선택 근거 부재** — 7수 중 왜 끝의 3수인지가 힌트2에만 존재. **수정 방향**: 수첩 자동 생성 합성 규칙 카드 문구를 "새가 남긴 높이, 오르골이 남긴 박자" → "**새의 노래는 끝의 세 걸음으로 저문다 — 새가 남긴 높이, 오르골이 남긴 박자**"로 보강하거나, 자물쇠 옆 새김(운지 대응표)에 음표 3개+꼬리 새김을 추가해 "끝에서 센다"를 시각 단서화.
4. **[대상: 퀴즈] S07 — 균형추 中 식별 단서 모호** — 각주 "빵 한 덩이=모래 반 주머니"만으로는 小·中·大 중 무엇이 '한 주머니'인지 미확정. **수정 방향**: 中 주머니에만 "한 주머니(표준)" 마대 각인을 추가하거나, 각주를 "빵 한 덩이 = 표준 주머니 반. 표준은 가운데 크기"로 확장. (오답 시 리셋 무벌점이라 치명은 아니나 단서 원칙 위반)
5. **[대상: 퀴즈] 부속표 3 — 소요시간 합계 오류** — 챕터별 소요의 실합은 130분인데 합계가 "약 120분"으로 표기. **수정 방향**: 목표가 120분이면 Ch4·Ch6에서 각 5분 감축(S12 시연 재생 속도 옵션 기본화, S19 판 선택 탭 초기 하이라이트)하고, 130분을 수용하면 합계를 "약 130분(±10)"으로 정정. 어느 쪽이든 표와 실합을 일치시킬 것.
6. **[대상: 스토리] §8 인수인계 메모 4번** — "포인터 드래그 기반" 문구를 "탭/길게 누르기 기본, 드래그는 선택적 보조 입력"으로 수정 (안건① 채택 반영).
7. **[대상: 스토리] 슬롯표 S14 + 챕터 개방 규칙** — S14 비고에 "2단계: 관찰(Ch4)/완결(T5 후)" 명기, §2 또는 §4에 "챕터 개방 조건은 핵심 슬롯 기준(Ch5 개방에 S14 완결 불포함)" 1줄 추가 (안건② 채택 반영).
8. **[대상: 스토리+퀴즈] 전시물 12점 목록 정본화** — 안건③ 권장안 적용: 조약돌 주머니를 12점에 포함, 레시피 카드를 목록 제외. 퀴즈 S28 목록·S01 체크리스트 연출·스토리 Ch1 본문("전시물 12점")에 동일 목록 사용. 부록 별항 구조 유지 시에는 스토리 본문을 "12점+부록 1점" 표기로 수정.
9. **[대상: 퀴즈, 경미] S28 힌트3** — "관찰만 하고 지나친 물건 둘(부는 뼈, 꿰맨 주머니)" 직접 지목은 '로그 재열람 유도만' 원칙의 경계선. "수첩에서 '교체 근거가 아직 안 붙은' 관찰 카드를 찾아라" 수준의 간접 문구로 완화 권장.

## 잘된 점 (스킬 추출 후보)

1. **S02↔S31 동일 UI 대칭 반전** — 같은 조작(카드 순서 배열)을 게임 최초와 클라이맥스에 재사용해 "같은 손짓으로 정반대 이야기를 배열"하게 하는 설계. 반전을 텍스트가 아니라 **조작의 기시감**으로 체감시키는 기법 — 범용 스킬로 추출 가치 높음.
2. **불일치 로그 축적 → S28 총회수 구조** — 위화감을 5건의 수첩 로그로 자동 축적하고, 반전 방아쇠 퀴즈가 "새 정보 주입 없이" 기존 로그만으로 풀리게 한 설계. "반전의 재료는 전부 플레이어 손에 있었다"를 시스템으로 보장.
3. **복선의 조작화** — S27 안경 착용 시 화면 블러(실명 도수 체험)처럼, 복선을 읽히는 게 아니라 **조작으로 겪게** 하는 처리. F16이 텍스트 한 줄이 아니라 UI 필터로 전달됨.
