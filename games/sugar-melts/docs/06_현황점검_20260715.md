## 3. 스토리 구조

### 3-1. 표면 이야기 vs 진짜 이야기
- **표면**: 관광객에게 파는 "헨젤과 그레텔" 동화 사적지 — 안내판 12점을 검증관이 확인하고 인증서에 서명하는 절차극.
- **진짜 이야기** (그레텔의 편지로 드러남, index.html:577 `LETTER` 변수):
  - 관리인 '할미' = 65년 전 이 집에서 겨울을 난 소녀, 즉 그레텔 본인.
  - 정원사(말 없는 노인) = 그녀의 오빠 헨젤. 그날 밤 우리 안에 숨어 전부 목격한 뒤 말을 잃음.
  - '아그네스'(동화 속 마녀)는 실존 인물이었고, 마을 사람들이 그녀를 가마에 밀어 넣어 죽였다. 문을 닫은 것은 어른들이 아니라 어린 그레텔의 손 — 어른들이 그 손을 걸쇠에 얹게 만들었다("네가 닫으면 이건 네 이야기가 된다").
  - 헨젤이 매일 새벽 집에 설탕을 새로 입히는 행위는 은폐가 아니라 "무덤을 닦는 일"이었다는 반전.
  - 3대 반전: ① 동화 = 위조된 알리바이 ② 할미 = 그레�텔 ③ 정원사 = 헨젤. (Ch9 이전까지 문제·힌트·오답 리액션에서 전혀 노출되지 않도록 설계·검증됨 — 4절 참조)

## 7. 알려진 이슈 / 확인이 필요한 사항

## 8. 수정 작업 실무 참고

- **문제(퀴즈/미션) 텍스트·정답·힌트 수정**: `index.html`에서 `defSlot({id:'S__'` 검색 → `desc`(문제문), `hints`(힌트 3단계 배열), `ans`(정답 표시용 텍스트) 필드 수정. 실제 정답 판정 로직은 같은 슬롯의 `build()` 함수 내부에 있으므로 `ans` 필드(표시용)와 판정 로직을 반드시 함께 확인·일치시킬 것.
- **스토리 대사·연출 수정**: 각 슬롯 `build()` 내부의 `say([{s:'화자',t:'대사'}])` 호출부, 방 소개는 `ROOMS`(506행), 챕터 인트로는 `CH_INTRO`(548행), 엔딩 본문은 `ENDINGS`(579행), 핵심 반전 편지는 `LETTER`(577행).
- **아이템 추가/변경**: `ITEMS` 객체(559행)에 정의 후, 실제 지급 지점(`gainItem('키')`)과 사용 지점(`hasItem`/`loseItem`)을 슬롯 코드에서 함께 연결해야 함.
- **수정 후 필수 절차**: ① `index.html` 수정 → ② 동일 내용을 `룸메이커_플랫폼/games/sugar-melts.html`에 복사 → ③ 사용자가 PC에서 룸메이커_플랫폼 폴더에서 `./publish.ps1 "메시지"` 실행해야 실제 배포 반영됨(샌드박스에서는 git push 불가).
