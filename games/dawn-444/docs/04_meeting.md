## 2. 게임의 뼈대 — SCENES 배열

게임 진행 전체는 HTML 안 `<script>`의 **`SCENES` 배열 하나**로 정의됩니다.
배열 순서 = 플레이 순서. 항목(장면)은 4종류:

| type | 역할 | 필수 필드 |
|---|---|---|
| `story` | 내레이션 화면 (프롤로그 등) | `ch, icon, text[], btn` (+선택 `wall`) |
| `chapter` | 챕터 표지 화면 | `ch, icon, desc` |
| `letter` | 문틀 글자 획득 화면 (챕터 클리어) | `ch, letter, cap` |
| `puzzle` | 문제 화면 | `ch, no, ttl, icon, dsc, kind, hints[]` (+선택 `pre[], wall, msgs[], after[]`) |

현재 흐름: `프롤로그(story)` → `1장(chapter)` → `문제1~4(puzzle)` → `글자(letter)` → `2장…` 반복 → `문제20` → 엔딩(코드로 처리).

### 공통 필드 사전 (puzzle 기준)

| 필드 | 뜻 | 비고 |
|---|---|---|
| `ch` | 챕터 번호 0~5 | 상단바 표기·배경색·파티클 색이 이 값을 따름 |
| `no` | 문제 번호 (1~20) | 화면에 "PUZZLE nn / 20"으로 표시 |
| `ttl` | 문제 제목 | |
| `icon` | 장면 이모지 | |
| `pre` | 문제 전 스토리 문단 배열 | HTML 태그 사용 가능 (`<em>강조</em>`) |
| `wall` | 벽 글씨/쪽지 상자 (파란 인용 박스) | `<br>`로 줄바꿈 |
| `msgs` | 목록 카드 (문자·거울·부재중 등) | `{t:'라벨', x:'내용'}` 배열 |
| `dsc` | 문제 지시문 | |
| `kind` | 문제 유형 (아래 3장) | |
| `hints` | 힌트 3개 배열 | **반드시 정확히 3개** |
| `after` | 정답 후 연출 문단 배열 | 스토리 감정선이 여기 담김 |

> 스토리만 바꾸고 싶다면: `pre`, `wall`, `after`, `story`의 `text`, `chapter`의 `desc`만 고치면 됩니다. 엔진은 건드릴 필요 없음.

## 3. 문제 유형(kind)별 명세와 수정법

### 3-1. `code` — 숫자 자물쇠
```js
kind:'code', len:3, ans:'716'
```
- `len` = 자릿수(입력 칸 수), `ans` = 정답 문자열. **자릿수와 ans 길이 일치 필수.**
- 수정: 두 값만 바꾸면 끝. 사용 문제: **1, 4, 6, 13, 17**

### 3-2. `text` — 텍스트 입력
```js
kind:'text', ans:['빛은등뒤에있다'], ph:'입력 안내문'
```
- 판정 규칙: 입력값에서 **공백과 문장부호( . , ! ? ' " · - — ~ 따옴표류)를 전부 제거**한 뒤 `ans` 배열과 비교.
- 따라서 `ans`에는 **공백 없는 형태**로 적을 것. 복수 정답 허용 시 배열에 추가: `ans:['정답a','정답b']`
- 사용 문제: **2, 3, 7, 9, 11, 19**

### 3-3. `order` — 순서 배열 (탭해서 순서 지정)
```js
kind:'order',
items:[ {ico:'🎂', lbl:'항목 이름', sub:'보조 설명'}, … ],
correct:[0,1,2,3,4]
```
- `items`는 **정답 순서대로** 적고, `correct`는 그 인덱스 순서(보통 `[0,1,2,…]`).
- 화면 표시 순서는 엔진이 자동으로 섞음(회전+뒤집기) — 신경 쓸 필요 없음.
- 전부 선택하면 자동 판정, 틀리면 자동 리셋. 사용 문제: **5, 14, 18**

### 3-4. `free` — 정답 없는 입력 (16번)
```js
kind:'free', ph:'안내문', btn:'보내기'
```
- 빈 칸만 아니면 무조건 통과. 실패 문구는 엔진 `checkFree()`의 `'빈 마음은 전송되지 않습니다.'`

### 3-5. 특수 유형 — 데이터가 아니라 **엔진 함수에 하드코딩**됨

이 4종은 SCENES의 텍스트(스토리·힌트)는 자유롭게 고쳐도 되지만, **정답 로직은 아래 함수를 직접 수정**해야 합니다.

| kind | 문제 | 로직 위치 (검색어) | 바꿀 값 |
|---|---|---|---|
| `tile` (8번) | 5×5 타일 밟기 | `puzzleInit` 안 `s.kind==='tile'` | 정답 좌표 `x===4&&y===3`, 표식 `marks={'1,1':'시소','4,1':'미끄럼','1,3':'그네'}` (좌표는 x=왼쪽부터, y=아래부터) |
| `scale` (10번) | 저울 균형 | `puzzleInit`의 추 목록 `ws=[['재능',3],…]` + `checkScale()` | 추 이름·무게, 정답 조합 `world`/`mine` 배열 |
| `stars` (12번) | 별자리 잇기 | 전역 `STARS` 배열 + `clickStar()` | 각 점의 `{r:순번(0=함정), x,y:위치%}`. 순번 개수 바꾸면 `starNext>7`의 7도 변경 |
| `seats` (15번) | 자리 배치 | `SEAT_NAMES` 배열 + `checkSeats()` | 이름 목록, 정답 조건 `N==='엄마'&&S==='은사'&&E==='준호'&&W==='나'` |
| `door` (20번) | 밀기/당기기 | `pushDoor()` / `pullDoor()` | 실패·성공 동작. `pullDoor`가 엔딩 호출 |

## 8. 문제 추가·삭제·순서 변경 절차

1. **추가**: SCENES 배열의 원하는 위치에 puzzle 객체 삽입 (§2 필드 + §3 kind 선택).
2. **번호 정리**: 이후 문제들의 `no`를 밀어서 연번 유지.
3. **총 개수 갱신**: §7 표의 "총 문제 수 표기" 4곳을 새 개수로 (예: 21문제면 `/21`).
4. **챕터 배분**: 챕터당 4문제 구조가 깨져도 동작엔 문제없음. 단 `letter` 장면은 챕터 마지막 문제 뒤에 둘 것.
5. **삭제**: 객체 삭제 후 2~3번 동일 진행. 복선 체인(§5)에 걸린 문제는 대체 설계 필수.
6. **정답 페이지**(`dawn-444-answers.html`)에 카드 추가/삭제.
7. 브라우저에서 처음부터 끝까지 1회 통주(通走) 테스트 → `./publish.ps1`.
